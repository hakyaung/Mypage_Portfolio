기본 스타일 시트

hn 태그가 각각 다르게 표시된 이유는 웹 브라우저 자체에 기본 스타일 시트가 내장되어 있기 때문
따라서 웹 브라우저에 내장된 기본 스타일 시트에 영향을 받아 크기와 굵기가 각각 다르게 표시됨

적용 우선순위와 개별성

몇 개의 태그는 이미 스타일이 적용된 상태로 표시됨
여기에 사용자가 별도로 스타일 속성을 정의하면 정의한 스타일 속성이 적용된 상태로 표시됨
즉 기본 스타일 시트에 정의된 스타일 속성보다 사용자가 정의한 스타일 속성이 우선 적용됨을 의미

h1 tag

같은 태그 요소의 선택자가 여럿일 때도 한 속성만 적용되는 이유
단계적 적용: 같은 태그에 여러 스타일이 적용되더라도 단계적으로 적용되어 결국 마지막에 영향을 주는 하나의 스타일만 적용된 다는 뜻
▼ ▼ ▼

specificity

개별성 규칙의 점수
선택자 예 점수
전체 선택자 * 0

important

단위

단위는 절대 단위와 상대 단위로 나뉨
절대 단위 어떤 환경이라도 동일한 크기로 보이는 단위
상대 단위 부모 요소 또는 웹 브라우저의 창 크기에 따라 상대적으로 결정되는 단위
절대 단위
절대 단위는 px(pixel) 한 가지
상대 단위
CSS에서 자주 사용하는 단위 단위는 다음과 같음
상대 단위
단위 설명
% 해당 속성의 상위 요소 속성값에 상대적인 크기를 가짐
em 부모 요소의 텍스트 크기에 상대적인 크기를 가짐
(만약 값을 2em 으로 지정했다면 자식 요소의 크기는 부모 요소의 font-size 속성값의 2배가 된다)
rem html 태그의 텍스트 크기에 상대적인 크기를 가짐
(1rem 은 html 태그의 텍스트 크기의 1배
1rem = 16px)
vw 뷰포트의 너비를 기준으로 상대적인 크기를 가짐
(웹 브라우저의 창의 너비가 뷰포트 너비가 됨
1vw는 뷰포트 너비의 1/100 크기를 의미)
vh 뷰포트의 높이를 기준으로 상대적인 크기를 가짐
(1vh는 뷰포트 높이의 1/100 크기를 의미)
font-size 80% | 16px(부모 font-size 크기) * 0.8 = 12.8px
font-size 80% | 12.8px(부모 font-size 크기) * 0.8 = 10.24px
font-size 2em | 16px(부모 font-size 크기) * 2em = 32px
font-size 2em | 32px(부모 font-size 크기) * 2em = 64px
font-size 1rem | 16px * 1rem = 16px
font-size 2rem | 16px * 2rem = 32px
font-size 1vw | 900px * 1/100 = 9px
font-size 2vw | 900px * 2/100 = 18px
font-size 1vh | 400px * 1/100 = 4px
font-size 2vh | 400px * 2/100 = 8px
